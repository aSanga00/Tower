		
	Tower DeffenceブレストMTG	
		
■	目標	
・	ユニットそれぞれに特性をもたせて、AIで動かす	
・	わかりやすくルールはなるべくシンプルに、雰囲気や操作によって得られるフィードバックが明快にする	
・	攻めるゲームにしたい	
		
		
■	参考	
	アークナイツ	
	→	古き良きタワーディフェンス
	→	アークナイツはコストを支払ってキャラを配置する→キャラを選んで向きを決める
		
■	内容について	
・	ユニット	
	→	ユニットには敵も味方も制限がある
	→	ユニットは特有のAIをもっていて、AIの行動部分が特性になる
	→	ユニットの特性は遠距離（攻撃・補助）・近距離・防御（タンク）・ヒーラー・ボマーなど種類がある
	→	ユニットは進行速度の設定がある
	→	ユニットの中には、拠点を破壊するのが専門がいても良い
	→	ユニットを出すためにはコストを消費する（敵も味方も）
	→	ユニットを出撃させるさいのコストは時間経過で回復していく
	→	ユニットには攻撃範囲の設定がある
	→	攻撃範囲のないタンクのユニットがいる
	→	遠距離ユニットの場合、銃は直線だけど、弓は山なりといった設定がある
	→	ユニットは索敵範囲をもっていて、攻撃できる範囲がある。
	→	ユニットの属性は一旦おいておく←ユニットの個性に意味合いをもたせる
	→	ユニットによって、特有のスキルがあってもよい
	→	リーダースキルはユーザーが選んで決める？←最後でもOK
・	拠点	
	→	ユニットと拠点は別物の設定
	→	拠点が一つだけだと面白みが少ないので、少なくとも2,3点は欲しい
	→	拠点の近くでないと自分のユニットが出せない
	→	ゲームのスタートに拠点の位置を決める。
	→	拠点の作成位置は限られている。
	→	拠点には本陣があって、そこが壊されるとゲーム終了となる要素あっても良い
		
・	パラメーター系	
	→	パラメータ：攻撃範囲・移動スピード・攻撃スピード・HP・コスト・攻撃力・索敵範囲
	→	レベルはバトル中に上がるのはなしにしたい（リスキー・不確定要素が多い）
	→	レベルアップはバトル終了後
		
		
・	ゲーム進行	
	→	攻撃範囲や移動可能を示すマス目がある
	→	移動はマス目ごと
	→	移動中もダメージは受けるし与えられる。
	→	段差の設定
	→	ユニットは移動の目的ポイントに移動する。
	→	移動が終了した時点で初めて行動する。
	→	移動中に迂回路を取れる状態であれば、ルート変更できる。
	→	ルートによって、攻められる拠点が違うのであれば、目的ポイントは変えられる。
	→	基本的に移動は最短距離を取る
	→	マス目に何体のユニットが置けるかは試してみる→ユニットを重ねることによって攻撃や補助の特性にバリエーションが出そう
	→	攻撃する際はマス目に止まって攻撃
	→	基本的に移動中に攻撃できるユニットがいると、立ち止まって攻撃する
	→	上記以外のパターンもいる→素通りタイプ（拠点しか攻撃しない）
		
・	味方ユニットの再利用について	
	→	ユニットが生存している限りは同ユニットは配置できない。
	→	ユニットが戦闘に負けた場合はクールタイムを設けて、再配置できるようにする
	→	出撃したユニットをキャンセルできるが、出撃コストが返ってこなかったりのデメリットがある。
